Cover Trainer

Die App erwartet den Ordner ./images neben der index.html. Darin liegen nummerierte Unterordner 1, 2, 3, … mit exakt image.png und optional cover.json.

So verwendest du die App

  1. Lege die App-Dateien in einen Ordner.
  2. Lege daneben den Ordner images an.
  3. Lege darin nummerierte Unterordner an: 1, 2, 3, …
  4. Lege in jedem Unterordner genau image.png ab.
  5. Optional ergänze cover.json.
  6. Starte index.html per VS Code Live Server.

Verhalten pro Ordner:

  • Ordner ohne image.png: Alert mit Fehlermeldung
  • image.png ohne cover.json: Editor zum Erstellen neuer Cover-Zonen
  • image.png mit cover.json: Viewer mit grauen Zonen zum Aufdecken/Abdecken
Information

Wie die App arbeitet

Funktionsweise

Die App liest nicht frei alle Unterordner aus, sondern prüft feste Pfade:

./images/1/, ./images/2/, ./images/3/, …

Pro Ordner werden diese festen Dateien erwartet:

  • image.png als Bilddatei
  • cover.json optional für vorhandene Cover-Zonen

Die Suche stoppt erst, wenn 100 Ordnernummern in Folge fehlen. Dadurch sind Lücken erlaubt, zum Beispiel 1, 2, 7, 18.

Fehlerquellen
  • Der Dateiname muss exakt image.png heißen. Image.png, image.PNG oder bild.png werden nicht erkannt.
  • Die Ordnernamen müssen rein numerisch sein: 1, 2, 3, …
  • Wenn mehr als 100 Nummern am Stück fehlen, endet der Scan vorher. Höhere Ordner werden dann nicht mehr angezeigt.
  • Die App muss über einen lokalen Webserver laufen, zum Beispiel VS Code Live Server. Per Doppelklick auf index.html funktioniert es nicht zuverlässig.
  • Ein fehlerhaftes cover.json öffnet den Editor statt des Viewers.
  • Browser- oder Server-Caching kann alte Daten zeigen; dafür gibt es den Button Liste aktualisieren.
  • Auf manchen Systemen ist die Schreibweise von Dateinamen case-sensitive.
